Checklist de Exportação: Replit para IDE Local (IA-Ready)
Este checklist garante que o seu projeto saia do Replit pronto para ser entendido por qualquer IDE de IA (Antigravity, Cursor, VS Code + Copilot).

1. Documentação de Contexto (Obrigatório)
 .agents: Arquivo na raiz com as "regras de ouro" do projeto (Stack, o que NÃO mexer, padrões de design).
 client/DESIGN_SYSTEM.md: Guia visual completo (cores, fontes, componentes, arredondamentos).
 README.md: Atualizado com o Sitemap (todas as rotas) e a lógica de dados (ex: LocalStorage).
 INSTALL.md: Passo a passo de como rodar npm install e npm run dev fora do Replit.
2. Configuração de Ambiente
 client/package.json: Criar um arquivo de dependências limpo apenas para o frontend.
 tsconfig.json: Garantir que os @/ aliases (caminhos absolutos) estejam configurados.
 .gitignore: Certificar que node_modules e pastas de cache não serão levadas.
 vite.config.ts: Verificar se os plugins necessários e o proxy (se houver) estão configurados para rodar localmente.
3. Assets e Metadados
 Favicon e Logos: Verificar se os arquivos físicos existem na pasta public/.
 index.html: Atualizar og:image e theme-color com os assets da marca.
 Mock Data: Garantir que as imagens de exemplo nos relatórios usem URLs externas (Unsplash) para o zip não ficar pesado.
4. Estrutura de Código
 Layout Modular: Separar Sidebar, Navbar e Containers em components/layout/.
 Páginas: Cada rota do App.tsx deve ter seu próprio arquivo em pages/.
 Animações: Se houver Framer Motion, garantir que os PageWrappers estejam no App.tsx.
 Clean Code: Remover console.logs desnecessários e comentários de código "morto" que podem confundir a IA.
5. Validação Final
 Check de Erros: Rodar o comando de verificação de tipos (tsc) para garantir que não há erros de importação.
 Checkpoint: Fazer o deploy/publish final no Replit para servir de "gabarito" visual.
 Zip Test: Se possível, baixar e rodar em uma pasta temporária para garantir que nada ficou de fora.
